## Pro Git

{{< keywordList >}}
{{< keyword >}}Scott Chacon {{< /keyword >}}
{{< keyword >}}Ben Straub{{< /keyword >}}
{{< /keywordList >}}

<article>
    <figure>
        <a href="https://git-scm.com/book/en/v2/" target="_blank" rel="noopener noreferrer" class="block w-44 md:w-56 mx-auto">
            <img src="pro-git.jpg" alt="Pro Git cover" class="nozoom block w-full h-auto rounded-lg">
        </a>
        <figcaption class="text-center">
            Pro Git, by Scott Chacon and Ben Straub
        </figcaption>
    </figure>
    <p class="text-justify">
        Version control is an essential part of software development, and <strong>Git</strong> is the de facto tool.
        <a href="https://git-scm.com/book/en/v2/" target="_blank" rel="noopener noreferrer">Pro Git</a>
        provides all the information needed to get started as a beginner, but it also covers advanced topics for experienced users. <br>
        I believe that even though 90% of the work can be done using fewer than 10 <code>git</code> commands, having a more in-depth understanding of Git provides the confidence to use it effectively and troubleshoot unexpected situations. (<code>reflog</code>, <code>bisect</code>, ...)
    </p>

</article>

## Ansible for DevOps

{{< keyword >}}Jeff Geerling{{< /keyword >}}

<article>
    <figure>
        <a href="https://www.ansiblefordevops.com/" target="_blank" rel="noopener noreferrer" class="block w-44 md:w-56 mx-auto">
        <img src="ansible-for-devops.jpg" alt="Ansible for DevOps cover" class="nozoom block w-full h-auto rounded-lg">
        </a>
        <figcaption class="text-center">
            Ansible for DevOps, by Jeff Geerling
        </figcaption>
    </figure>
    <p class="text-justify">
        <strong>Ansible</strong> is a wonderful tool for automating almost everything server-related.
        <a href="https://www.ansiblefordevops.com/" target="_blank" rel="noopener noreferrer">Ansible for DevOps</a>
        guides the reader from running ad-hoc Ansible commands to writing playbooks and organizing solutions into roles.
        The author, Jeff Geerling, is well known in the Ansible community, and he maintains several widely used Ansible roles on GitHub. <br>
        I believe that knowing the basics of Ansible can be very useful for anyone who enjoys automation.
    </p>
</article>
